# Contributing

Thanks for considering a contribution. This skill is built from real client work, and the goal is to keep it grounded in actual practice, not theory.

## What kind of contributions are wanted

- **New cleaning patterns** for pandas that handle a specific messy-data case you have seen in client work
- **New client-type playbooks** if you have done automation work in a vertical not covered (logistics, healthcare, hospitality, manufacturing, agriculture)
- **New dashboard stack recipes** beyond Metabase, Streamlit, and Next.js
- **New automation recipes** that solve a real recurring task
- **Fixes** to existing patterns if they break on real-world data

## What is NOT wanted

- Theoretical patterns you have not used on a real client project
- Heavy frameworks or abstractions
- Fillers and wordy explanations. Keep additions tight.
- Marketing copy about "leveraging" or "unlocking" anything

## How to contribute

1. Fork the repo
2. Create a branch: `git checkout -b add-retail-playbook`
3. Make your change in `SKILL.md` or add a new file in the right section
4. Keep your writing short and direct
5. Open a pull request

In your PR description, include:

- What you added
- What client project or context this came from (anonymized is fine)
- Why the existing skill did not cover it

## Style guide

- No em dashes. Use regular hyphens or commas.
- No filler phrases ("in today's fast-paced world", "game-changer", "robust solution")
- Short paragraphs. Often one or two sentences.
- Real numbers. No "countless" or "numerous".
- Show code, not paragraphs about code.
